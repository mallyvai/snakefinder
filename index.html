snakefinder by mallyvai

Welcome!

Snakefinder is a Simple Python Source search engine prototype. Index a large Python codebase, then quickly query it in a hierarchical way. Discover structure, classes, and functions much more rapidly than with grep, awk, or (gasp) perl :-)

Take a look at my well-received presentation, which explains rationale, performance characteristics, and ‘shortcomings’ (all of which i have in an a few experimental branches that I need to get around to merging with master at some point). If there is demand ( read: emails ), I’ll happily move faster on the merging.

The presentation also has the BNF for the query language, if you're interested.

Or, read the paper (it's not too dense, I promise :-) )

Usage

1) Feed your tasty codebase to Snakefinder:

`python indexer.py /dir/to/codebase`

You should notice that you have an 'index.pkl' file in the directory.

2) Query it using python’s regex syntax augmented with Snakefinder’s hierarchical mechanisms.

`python query.py`

BNF for query syntax:

<regex> ::= [python regex without spaces]

<type> ::= module | file | class | def

<part> ::= <type>=<regex>,<part> | <type> > | <type> >> | <type>

Operators

“a >> b” means “b has some ancestor a”

“a > b” means “a has a child b”

“a,b” means “a or b”

Sample queries

class=smart.+ | dumb.+ > func=spider

means "find any class beginning with ‘smart’ or ‘dumb’ that has a child function named ‘spider’ "

file=libpython.+ > class=.+

means "display all top-level classes in files named ‘libpython’ "

file=.+ >> class=.+

will display all classes in all files.
